# 制作一个可以测试车机休眠唤醒的 程序， 架构仿造 `/home/an/workspace/python/meltano` 

1. 目前支持cli, 保留未来支持gui的可能性

## 功能：
1. 发送命令让车机休眠
   + 支持多种休眠方式
   + 暂时实现其中一种：通过发送 CAN UDS 的方式让车机休眠 , 保留支持其他方式的可能性（可扩展）
   + 通过cli 参数的方式制定休眠方式
   + 通过ini 的方式指定 uds id, value 等参数
2. 发送唤醒命令让车机唤醒
   + 支持多种唤醒方式
   + 暂时实现其中一种：通过发送 CAN UDS 的方式让车机唤醒 , 保留支持其他方式的可能性（可扩展）
   + 通过cli 参数的方式制定唤醒方式
   + 通过ini 的方式指定 uds id, value 等参数
3. 暂时通过config.ini 来设定 物理地址 0x773 , 响应地址 0x7b3, 功能地址 0x7df
4. 默认 config.ini ， 但是可以通过cli参数的方式指定别的ini文件
5. 多种方式检测当前是否处于休眠状态
   + 暂时实现看MCU的话：通过串口来看是否休眠成功(串口连着MCU) 超过 500ms 没数据 → 认为睡了(除非你有更好的办法)
   + 暂时实现看SOC的话：通过adb 连接看 adb 连接是否还在来判定睡没睡
6. 可以通过cli 指定数字 作为循环 休眠唤醒的次数 , 比如 10000 就休眠唤醒10000次
7. 结束后输出报告， 报告格式为 result_yyyy_MM_DD.md

## 约束
1. 核心功能在 src/core 目录， 包括状态机， timeout啥啥
2. cli 胶水层在 src/cli 目录
3. 所有接口要有中文注释， 本项目使用 MkDocs + mkdocstrings , 接口注释使用 Google 风格 docstring
