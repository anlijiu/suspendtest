# 报告格式说明

报告由 `ReportService` 生成，默认输出文件名为：

```text
result_YYYY_MM_DD.md
```

也可通过 CLI 的 `--report` 指定输出路径。

## 报告结构

报告为 Markdown 文档，包含以下部分：

1. `# 休眠唤醒测试报告`
2. `## 执行信息`
   - 开始时间
   - 结束时间
   - 配置文件路径
   - 总轮次/成功轮次/失败轮次
3. `## 配置摘要`
4. `## 每轮结果`（表格）
5. `## 失败详情`

## 每轮结果表字段

| 字段 | 说明 |
| --- | --- |
| 轮次 | 从 1 开始 |
| 成功 | `✅`/`❌` |
| 休眠命令 | 发送结果消息 |
| MCU休眠 / SOC休眠 | 检测结果和耗时 |
| 唤醒命令 | 发送结果消息 |
| MCU唤醒 / SOC唤醒 | 检测结果和耗时 |
| 失败原因 | 多个原因用 `<br>` 拼接 |

## 成功判定逻辑

一轮测试只有在以下条件同时满足时才记为成功：

1. 休眠命令发送成功
2. 唤醒命令发送成功
3. 所有检测器在休眠阶段均达到目标
4. 所有检测器在唤醒阶段均达到目标

否则该轮标记为失败，并记录错误原因。

## 建议的报告归档策略

- CI/自动化场景：使用 `--report artifacts/<name>.md`
- 本地调试场景：保留默认命名，按日期归档
- 当轮次较多时，建议配合日志系统记录更细粒度原始数据
